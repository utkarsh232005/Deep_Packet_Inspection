## 📋 WHAT THE ENGINE DOES

INPUT ➜ PROCESSING ➜ OUTPUT

- Reads PCAP files (network packet captures)
- Parses packets (Ethernet, IPv4, TCP, UDP layers)
- Extracts SNI from HTTPS/TLS to identify apps
- Classifies 25+ applications (YouTube, Netflix, Facebook, etc.)
- Applies blocking rules:
  • Block by source IP address
  • Block by destination port
  • Block by application
  • Block by domain (with wildcard support)
- Writes filtered packets to output PCAP
- Generates statistics reports
- Uses multi-threading for high performance:
  • Load Balancers distribute packets
  • FastPath Processors analyze in parallel
  • Maintains TCP/UDP connection state

## 🎯 EXAMPLE USAGE

### Example 1: Block YouTube and Netflix
```bash
# Create a Java file: BlockApps.java
import com.dpi.engine.DPIEngine;
public class BlockApps {
    public static void main(String[] args) throws Exception {
        DPIEngine engine = new DPIEngine(new DPIEngine.Config());
        engine.blockApp("YOUTUBE");
        engine.blockApp("NETFLIX");
        engine.processFile("input.pcap", "output.pcap");
        System.out.println(engine.generateReport());
    }
}

# Compile and run
javac -cp target/packet-analyzer-1.0.0-jar-with-dependencies.jar BlockApps.java
java -cp .:target/packet-analyzer-1.0.0-jar-with-dependencies.jar BlockApps
```

### Example 2: Block Specific IPs and Domains
```bash
import com.dpi.engine.DPIEngine;
public class BlockIPsAndDomains {
    public static void main(String[] args) throws Exception {
        DPIEngine engine = new DPIEngine(new DPIEngine.Config());
        engine.blockIP("192.168.1.100");
        engine.blockIP("10.0.0.50");
        engine.blockDomain("facebook.com");
        engine.blockDomain("*.tiktok.com");  // Wildcard support
        engine.processFile("input.pcap", "output.pcap");
    }
}
```

## ⚙️ COMMAND-LINE OPTIONS

java -cp target/packet-analyzer-1.0.0-jar-with-dependencies.jar \
    com.dpi.engine.DPIEngineMain <input> <output> [options]

Options:
  -lbs N       Number of load balancers (default: 2)
  -fps N       FastPath processors per LB (default: 2)
  -queue N     Queue size (default: 10000)
  -rules FILE  Rules configuration file
  -v           Verbose logging

Threading Presets:
  Light:    -lbs 1 -fps 2    (2 threads)
  Medium:   -lbs 2 -fps 4    (8 threads)
  Heavy:    -lbs 4 -fps 4    (16 threads)
  Extreme:  -lbs 8 -fps 8    (64 threads)

## 🎓 ARCHITECTURE OVERVIEW

Input PCAP
    ↓
[Reader Thread]
    ↓
[Input Queue]
    ↓
[Load Balancer 1]  [Load Balancer 2]
    ↓                  ↓
[FP#0][FP#1]     [FP#2][FP#3]
    ├─────────────────────┤
            ↓
      [Output Queue]
            ↓
    [Writer Thread]
            ↓
       Output PCAP

Each FastPath Processor:
- Maintains connection table
- Extracts SNI from TLS
- Classifies applications
- Applies blocking rules
- Thread-safe operations
